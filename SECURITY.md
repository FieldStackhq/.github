# Security Policy

## Reporting a Vulnerability

FieldStack takes the security of our software and services seriously. If you discover a security vulnerability in any FieldStack repository, please report it responsibly.

### How to Report

**Email:** security@fieldstack.com

Please include:
- A description of the vulnerability
- - Steps to reproduce the issue
  - - The potential impact
    - - Any suggested remediation (optional)
     
      - ### What to Expect
     
      - - **Acknowledgment:** We will acknowledge receipt within 2 business days.
        - - **Assessment:** We will investigate within 5 business days.
          - - **Resolution:** Confirmed vulnerabilities resolved within 30 days.
            - - **Disclosure:** We coordinate timing with the reporter.
             
              - ### Scope
             
              - This policy applies to all repositories under the [FieldStackhq](https://github.com/FieldStackhq) organization.
             
              - ### Out of Scope
             
              - - Social engineering attacks
                - - Denial of service attacks
                  - - Issues in third-party dependencies (report to respective maintainers)
                   
                    - ## Supported Versions
                   
                    - Security updates are provided for the latest release of each active product.
                   
                    - ## Security Standards
                   
                    - All FieldStack repositories follow these security practices:
                    - - Branch protection with required reviews and status checks
                      - - Squash-merge only for clean, auditable history
                        - - Actions restricted to trusted sources with SHA pinning
                          - - Conventional commits for traceable change history
